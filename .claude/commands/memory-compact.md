# /memory-compact

这个命令保留原名称以兼容已有使用习惯，实际整理的是 `framework/` 和 `repos/` 单树知识目录。

建议每月运行一次，或 `CLAUDE.md` 新增规则累计达到 5 条时运行。

## 硬约束

- 只整理本仓 `CLAUDE.md`、`CONTRIBUTING.md`、`framework/` 和 `repos/`；当前机器信息只在 ignored `local/` 中检查，不复制到正式知识。
- 禁止写系统、全局或个人 memory。
- 每一步删改都先展示给用户；不静默移动页面。
- 合并或拆分不能丢语义，旧链接必须在同一次修改中更新。
- 不按角色建立 PM/QA/Dev 错题本；错题只按根因所在主题、代码模块或模型归类。
- 不引入固定开场仪式、多角色辩论或每次任务都要背诵的流程税。

## 1. 健康度快照

```powershell
(Get-Item CLAUDE.md).Length
python tools/check_knowledge_tree.py
rg --files framework repos -g "*.md" | Measure-Object
```

记录：

- `CLAUDE.md` 大小；
- 检查错误和提醒数；
- 超过 300 非空行或 16 KiB 的页面；
- 超过 7 个普通页面或 20 个分类页面的目录；
- “处理中”“待归类”错题数量。

## 2. 整理 `CLAUDE.md`

- 每条硬规则都问：过去一个月是否真的拦住过问题，是否必须在开工时读取。
- 只针对一个主题、仓库、代码模块或模型的规则下沉到最近目录。
- 同主题多条合成一句“触发条件 → 必做/禁止 → 详细链接”。
- 完整原因、案例和长日志不留在 `CLAUDE.md`。

## 3. 整理普通知识页面

- 300 非空行或 16 KiB：检查是否混入两个独立主题。
- 500 非空行或 32 KiB：必须按语义拆分，或在 `_index.md` 写明暂不拆的原因和复核日期。
- 一个目录直接放到第 8 个普通页面：建立 `guides/`、`incidents/` 或明确业务分类。
- 一个分类目录超过 20 篇当前有效页面：继续按稳定问题主题分类。
- 不按固定行数机械切文章。

## 4. 整理错题

- 抽查最近和仍处于“处理中”“待归类”的错题。
- 已验证的补齐验证证据；已形成稳定规则的标成“已提炼”并增加目标链接。
- 一件事故只能有一篇完整正文；重复页面合并后更新所有入口。
- 历史页面保留在最近目录并标“仅历史”，不默认读取。

## 5. 收尾

```powershell
python tools/check_knowledge_tree.py
git diff --check
```

汇报本次：拆分多少页面、合并多少重复内容、更新多少索引、解决多少待归类错题，以及仍需人工判断的问题。
