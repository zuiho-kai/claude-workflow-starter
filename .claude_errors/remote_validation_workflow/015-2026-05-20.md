## 2026-05-20 — 同一类远端节点不能共享路径记忆

**症状**：用户让我到 `ssh root@47.79.124.13 -p 31140` 跑验证，我套用了旧记录里 `root@106.15.124.84 -p 31140` 的 `/home/wzr/wt-*` 路径假设。实测 47.79 这台机器 `/home/wzr` 存在但没有记录里的 worktree/venv，`/data/wzr` 不存在；可用种子 repo/venv 主要在 `/root/d00806799`、`/root/hsliu`、`/root/z00586778`。用户提醒“remote server 不是记录了，在 /home/wzr 没有就更新一下”，随后又明确“每个机器文件路径都不一样，你分别记录”。我又犯了第二个错误：看到 `/home/wzr` 没有现成 repo 后，擅自切到 `/root/d00806799` 建 worktree，而不是遵从用户指定的 `/home/wzr` 根目录。

**根因**：
- 把“端口一样 / hostname 历史看起来一样”误当成同一套目录布局。
- 远端记录按“节点 B”聚合，没有按 `host:port` 分层；导致旧路径从 B2 泄漏到 B1。
- 发现 `/home/wzr` 不成立后，没有第一时间把这个事实写回机器专属记录。

**已确认的机器专属事实**：
- `root@47.79.124.13 -p 31140`：`/home/wzr` 存在；当前已创建 `/home/wzr/vllm-omni` 和 `/home/wzr/wt-codex-hunyuanimage3-step-batch`；`/data/wzr` 不存在；`/root/d00806799/vllm-omni` 只作为种子 repo/历史可读源；可只读使用 `/root/hsliu/.venv` 或 `/root/z00586778/.venv` 做环境验证，但不要改共享依赖。
- `root@106.15.124.84 -p 31140`：旧记录路径在 `/home/wzr/...`，未在 2026-05-20 对同一目标复测前，不能套给 47.79。
- `root@106.15.124.84 -p 31449`：旧记录稳定根在 `/data/wzr`，不是 `/home/wzr`。

**硬规则**：
1. 远端路径记忆的 key 必须是完整 `user@host:port`，不是“节点名”或“用途名”。
2. 用户指定根目录时，根目录是约束，不是候选；如果里面没有 repo，就在该根目录创建/更新 repo 或 worktree，不要擅自切到另一个根。
3. 每次发现路径事实变化，立刻按机器小节写入记录：`hostname`、repo 根、venv、模型缓存、不可用路径。
4. 旧记录只可作为候选；新机器第一次使用时必须用一个聚合脚本确认 `pwd`/`hostname`/`test -d`/`git remote -v`，然后再跑验证。
5. 不同机器之间禁止传播 worktree 路径。`/home/wzr/wt-*`、`/data/wzr/*`、`/root/*` 都必须标明属于哪台 `host:port`。
